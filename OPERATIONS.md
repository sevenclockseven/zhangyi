## 1. 系统概述

### 1.1 系统定位

易记是一套面向**个人代账会计**的轻量级代理记账系统。每个代理客户对应一个独立账套，账套之间互不干扰。

### 1.2 核心能力

| 模块 | 功能 |
|------|------|
| 多账套管理 | 每个客户独立账套，支持行业模板一键初始化 |
| 科目管理 | 树形科目表，7个辅助核算维度，模板同步 |
| 凭证管理 | 录入、审核、记账、反审核、反记账、作废、恢复、批量操作 |
| 账簿查询 | 总账、现金日记账、银行日记账、多栏账 |
| 报表中心 | 资产负债表、利润表、现金流量表、费用统计、应收应付账龄、自定义报表 |
| 设备管理 | 资产分类、卡片管理、折旧计提（直线法）、自动凭证 |
| 期末处理 | 自动结转损益、期末结账、反结账 |
| 系统管理 | 用户管理、账套权限、操作日志、备份恢复 |

### 1.3 技术架构

- **前端**：Vue 3 + Element Plus，单页应用，嵌入 Go 二进制
- **后端**：Go (Gin)，RESTful API
- **数据库**：SQLite（默认零配置）/ PostgreSQL（生产环境可选）
- **部署**：Docker Compose 一键部署，或直接运行二进制

## 2. 部署与配置

### 2.1 Docker Compose 部署（推荐）

创建 `docker-compose.yml`：

```yaml
services:
  zhangyi:
    image: ghcr.io/sevenclockseven/zhangyi:latest
    container_name: zhangyi
    restart: unless-stopped
    ports:
      - "8080:8080"
    volumes:
      - zhangyi-data:/app/data
      - zhangyi-backups:/app/backups
    environment:
      - TZ=Asia/Shanghai
      # 必填：安全配置
      - JWT_SECRET=your-random-secret-key-here
      - ADMIN_PASSWORD=your-admin-password
      # 可选：CORS白名单（逗号分隔）
      # - CORS_ORIGINS=https://your-domain.com
      # 可选：切换PostgreSQL
      # - DB_DRIVER=postgres
      # - DB_DSN=host=postgres user=zhangyi password=*** dbname=zhangyi port=5432 sslmode=disable
      # 可选：备份周期
      # - BACKUP_SCHEDULE=24h
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/api/health"]
      interval: 30s
      timeout: 3s
      retries: 3

volumes:
  zhangyi-data:
  zhangyi-backups:
```

启动命令：

```bash
docker compose up -d
```

### 2.2 环境变量一览

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `8080` | 服务端口 |
| `JWT_SECRET` | 随机生成 | JWT签名密钥，**生产环境必填** |
| `ADMIN_PASSWORD` | 随机生成 | 管理员初始密码，**生产环境必填**，首次登录后请修改 |
| `CORS_ORIGINS` | `localhost:8080` | CORS允许源，逗号分隔 |
| `DB_DRIVER` | `sqlite` | 数据库驱动：`sqlite` / `postgres` |
| `DB_DSN` | `data/zhangyi.db` | 数据库连接串 |
| `BACKUP_SCHEDULE` | `24h` | 备份周期：`disabled` / `hourly` / `6h` / `24h` |
| `BACKUP_DIR` | `backups` | 备份文件目录 |

> 首次启动时若未设置 `JWT_SECRET` 和 `ADMIN_PASSWORD`，系统会随机生成并打印在日志中，请及时记录。

### 2.3 Windows 本地运行

直接双击 `zhangyi.exe` 即可启动，默认使用 SQLite。

如需切换 PostgreSQL，创建 `start.bat`：

```bat
@echo off
set DB_DRIVER=postgres
set DB_DSN=host=localhost user=zhangyi password=你的密码 dbname=zhangyi port=5432 sslmode=disable
zhangyi.exe
```

### 2.4 Mac/Linux 本地运行

```bash
chmod +x zhangyi
./zhangyi
```

## 3. 登录与安全

### 3.1 登录

1. 打开浏览器访问 `http://localhost:8080`
2. 输入用户名和密码
3. 点击「登录」

### 3.2 首次登录

- 管理员账号：`admin`
- 密码：启动时随机生成，查看容器日志获取
- **登录后请立即修改默认密码**

### 3.3 修改密码

1. 点击右上角用户名 → 「修改密码」
2. 输入原密码和新密码
3. 点击「确定」

### 3.4 用户角色

| 角色 | 权限 |
|------|------|
| 管理员（admin） | 全部功能，用户管理，系统设置 |
| 普通用户（user） | 凭证录入、报表查看等日常操作，受账套权限控制 |

### 3.5 账套权限

管理员可在「系统设置 → 账套权限」中为用户分配账套访问权限：

| 权限级别 | 说明 |
|----------|------|
| 完全控制 | 可查看和修改该账套所有数据 |
| 只读 | 仅可查看，无法录入或修改 |

## 4. 工作台（仪表盘）

登录后默认进入工作台，展示系统概览：

- **账套总数**：当前管理的账套数量
- **本月凭证**：当月所有账套的凭证总数
- **待审核**：状态为「草稿」的凭证数量
- **已审核未记账**：已审核但未记账的凭证数量
- **快捷操作**：新建账套、新建凭证、查看报表

## 5. 账套管理

### 5.1 查看账套列表

点击左侧菜单「账套管理」，显示所有账套：

- 账套编码、客户名称、行业、纳税人类型、启用期间、状态
- 每行可快速跳转至该账套的「科目」或「凭证」

### 5.2 新建账套

1. 点击「新建账套」
2. 填写信息：
   - **客户名称**（必填）：客户的公司名称
   - **行业类型**（必填）：可多选，支持制造业/零售业/服务业/建筑业/房地产业/运输业/农业
   - **纳税人类型**：小规模 / 一般纳税人
   - **会计准则**：小企业会计准则 / 企业会计准则
   - **启用期间**（必填）：选择开始记账的年月
   - **联系人**、**电话**：可选
3. 点击「确定」

> 创建后系统会根据行业模板自动生成科目表，可在科目管理中调整。

### 5.3 启用/停用账套

在账套列表中，点击状态开关即可切换账套的启用/停用状态。

### 5.4 删除账套

点击「删除」按钮，确认后删除。**删除操作不可恢复，请谨慎操作。**

### 5.5 账套切换

页面顶部有全局账套选择器（下拉框），选择后所有页面的数据自动切换到对应账套。切换时系统会自动取消进行中的旧请求，确保数据一致性。

## 6. 科目管理

### 6.1 查看科目表

点击左侧菜单「科目管理」，以树形结构展示当前账套的科目表：

- 一级科目以粗体显示
- 点击父科目可展开/折叠子科目
- 支持按编码或名称搜索

### 6.2 新建科目

1. 点击「新建科目」
2. 填写信息：
   - **编码**（必填）：科目编码，如 `1001`、`100101`
   - **名称**（必填）：科目名称
   - **方向**（必填）：借方 / 贷方
   - **上级科目**：选择父科目（不选则为一级科目）
   - **辅助核算**：可多选（客户/供应商/部门/项目/员工/仓库/银行账户）
3. 点击「确定」

### 6.3 编辑/删除科目

- 点击操作列的「编辑」修改科目信息
- 点击「删除」移除科目（有下级科目或已使用时不可删除）

### 6.4 启用/停用科目

点击状态开关切换科目的启用/停用状态。停用的科目在凭证录入时不可选择。

### 6.5 同步模板

点击「同步模板」按钮，可将行业模板中的标准科目同步到当前账套。已存在的科目不会重复创建。

## 7. 期初余额

### 7.1 功能说明

期初余额用于在账套启用时录入各科目的期初借贷方余额，确保借贷平衡。

### 7.2 操作步骤

1. 点击左侧菜单「期初余额」
2. 选择账套（顶部下拉框）
3. 在表格中填写每个末级科目的期初借方或贷方余额
4. 系统自动计算合计行，检查借贷是否平衡
5. 点击「保存」

### 7.3 试算平衡

页面底部显示试算平衡结果：
- 借方合计 = 贷方合计 时显示「平衡」
- 不平衡时显示差额，需调整后才能保存

### 7.4 导入/导出

- **导出**：点击「导出CSV」将期初余额导出为 CSV 文件
- **导入**：点击「导入CSV」上传 CSV 文件批量导入期初余额

## 8. 凭证管理

### 8.1 查看凭证列表

点击左侧菜单「凭证管理」，显示当前账套的凭证列表：

- **筛选条件**：凭证状态、日期范围、关键字
- **列表字段**：凭证号、日期、类型、摘要、借方合计、贷方合计、状态、制单人

### 8.2 新建凭证

1. 点击「新建凭证」
2. 选择凭证类型：记账凭证 / 收款凭证 / 付款凭证
3. 填写日期
4. 添加凭证明细行：
   - 选择科目（支持编码/名称搜索）
   - 填写借方或贷方金额
   - 填写摘要
   - 如科目启用了辅助核算，需选择辅助项
5. 系统自动计算借贷合计，实时检查借贷平衡
6. 点击「保存」

> **快捷键**：在明细行中按 Enter 可快速跳转到下一行。

### 8.3 凭证状态流转

```
草稿(draft) → 审核(reviewed) → 记账(posted)
   ↑              ↓                 ↓
   ←── 反审核 ←── ←── 反记账 ←──
```

| 操作 | 说明 | 权限要求 |
|------|------|----------|
| 审核 | 确认凭证无误 | 只读用户不可操作 |
| 反审核 | 撤回审核（仅已审核状态） | 只读用户不可操作 |
| 记账 | 登记入账（仅已审核或草稿状态） | 只读用户不可操作 |
| 反记账 | 撤回记账（仅已记账状态） | 只读用户不可操作 |
| 作废 | 作废凭证（仅草稿状态） | 只读用户不可操作 |
| 恢复 | 恢复已作废凭证 | 只读用户不可操作 |

### 8.4 批量操作

1. 在列表中勾选多条凭证
2. 点击「批量审核」或「批量记账」

### 8.5 导出凭证

点击「导出」按钮，将当前筛选条件下的凭证导出为 CSV 文件。

### 8.6 使用模板

- **保存为模板**：在新建凭证时点击「保存为模板」，输入模板名称
- **加载模板**：在新建凭证时点击「加载模板」，选择已有模板自动填充

## 12. 自定义报表

### 12.1 功能说明

自定义报表允许用户通过公式定义个性化报表。

### 12.2 公式说明

| 函数 | 含义 | 参数 |
|------|------|------|
| `JE('科目编码', '方向')` | 本期发生额 | 科目编码 + 借/贷 |
| `QM('科目编码', '方向')` | 期末余额 | 科目编码 + 借/贷 |
| `QC('科目编码', '方向')` | 期初余额 | 科目编码 + 借/贷 |
| `JL('科目编码', '方向')` | 本年累计发生额 | 科目编码 + 借/贷 |

支持运算符：`+`、`-`

示例：
- `QM('1001', '借')` — 库存现金期末余额
- `JE('6001', '贷') - JE('6401', '借')` — 收入减成本

### 12.3 操作步骤

1. 点击「报表中心」→「自定义报表」标签
2. 点击「新建报表」
3. 定义报表行：
   - 行标签（如：一、营业收入）
   - 公式（如：`JE('6001', '贷')`）
   - 层级（用于缩进显示）
4. 点击「保存」
5. 选择期间后点击「运行」查看结果

## 13. 设备管理（固定资产）

1. 点击左侧菜单「设备管理」→「分类」标签

   - 分类编码、名称

|------|------|

| 已闲置（idle） | 停止折旧 |

### 13.4 折旧计提

1. 点击「折旧」标签
2. 选择计提期间
3. **试算**：点击「计算折旧」查看本月应计提金额（不执行）
4. **执行**：点击「计提折旧」正式执行，系统自动生成折旧凭证

### 13.5 资产台账

点击「台账」标签，查看按分类汇总的资产统计：
- 资产数量、原值合计、累计折旧、净值

### 13.6 变动记录

点击「变动记录」标签，查看所有资产的变动历史（购入、折旧、状态变更等）。

### 13.7 导入/导出

- **导出**：将资产卡片导出为 CSV 文件
- **导入**：通过拖拽或点击上传 CSV 文件批量导入资产

CSV 导入格式要求：

| 字段 | 说明 | 必填 |
|------|------|------|
| 编码 | 资产编码 | 是 |
| 名称 | 资产名称 | 是 |
| 分类 | 分类名称（自动匹配分类ID） | 是 |
| 原值 | 购置原值 | 是 |
| 购置日期 | YYYY-MM-DD 格式 | 否 |
| 使用部门 | 部门名称 | 否 |
| 使用人 | 员工姓名 | 否 |
| 存放位置 | 物理位置 | 否 |
| 来源 | 外购/自建/捐赠等 | 否 |
| 状态 | 使用中/已报废/已闲置 | 否 |

## 14. 期末处理

### 14.1 功能说明

期末处理用于完成会计期间的结转和封账操作。

### 14.2 操作流程

1. 点击左侧菜单「期末处理」
2. 查看当前状态：
   - 当前期间
   - 未记账凭证数量
3. 按顺序执行以下操作：

#### 第一步：结转损益（自动）

点击「自动结转」，系统自动将损益类科目余额结转至本年利润。

- 生成一张结转凭证
- 借方：费用类科目
- 贷方：收入类科目
- 差额记入本年利润

#### 第二步：期末结账

点击「期末结账」，系统执行：

1. 检查所有凭证是否已记账
2. 锁定当前期间（不可再录入该期间凭证）
3. 自动生成下一期间的期初余额

#### 反结账（如需修改）

点击「反结账」可撤销结账操作，重新开放该期间。

> 注意：反结账会删除下一期间的自动生成期初余额。

## 16. 常见问题

### Q1: 忘记管理员密码怎么办？

删除 SQLite 数据库文件（`data/zhangyi.db`），重启服务后系统会重新生成随机密码。

### Q2: 如何切换到 PostgreSQL？

1. 创建 PostgreSQL 数据库：`CREATE DATABASE zhangyi;`
2. 设置环境变量：
   ```
   DB_DRIVER=postgres
   DB_DSN=host=localhost user=zhangyi password=*** dbname=zhangyi port=5432 sslmode=disable
   ```
3. 重启服务

### Q3: 凭证借贷不平衡怎么办？

系统会实时检查借贷平衡，不平衡时无法保存。请检查各明细行的借方和贷方金额。

### Q4: 期末结账后发现错误怎么办？

在「期末处理」中点击「反结账」，修改凭证后重新结账。

### Q5: 如何备份数据？

方式一：在「系统设置 → 备份」中手动创建备份。
方式二：设置 `BACKUP_SCHEDULE=24h` 启用自动备份。

### Q6: 只读用户能做什么？

只读用户可以查看凭证、报表、账簿等数据，但无法创建、修改或删除任何数据。

### Q7: 切换账套时数据会混乱吗？

不会。切换账套时系统会自动取消所有进行中的请求，确保数据一致性。

### Q8: 如何导出数据？

系统各模块均提供 CSV 导出功能：
- 凭证管理：导出凭证列表
- 报表中心：导出报表数据
- 辅助核算：导出辅助项
- 设备管理：导出资产卡片
- 期初余额：导出余额表
