## 一、环境准备

### 1.1 安装 Go

**Windows:**
1. 访问 https://go.dev/dl/
2. 下载 `go1.21.x.windows-amd64.msi`
3. 双击安装，默认路径 `C:\Program Files\Go`
4. 打开 CMD 验证：
```cmd
go version
# 输出: go version go1.21.x windows/amd64
```

**Linux:**
```bash
wget https://go.dev/dl/go1.21.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.21.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc
go version
```

### 1.2 安装 Node.js

1. 访问 https://nodejs.org/
2. 下载 LTS 版本（20.x）
3. 安装时勾选"Add to PATH"
4. 验证：
```cmd
node -v
# 输出: v20.x.x

npm -v
# 输出: 10.x.x
```

### 1.3 安装 Git

1. 访问 https://git-scm.com/
2. 下载对应平台版本
3. 安装后验证：
```cmd
git --version
# 输出: git version 2.x.x
```

### 1.4 安装 GCC（CGO 编译需要）

SQLite 依赖 CGO，需要 GCC 编译器。

**Windows:**
1. 下载 MinGW-w64: https://www.mingw-w64.org/
2. 或者安装 TDM-GCC: https://jmeubank.github.io/tdm-gcc/
3. 安装后确保 `gcc` 在 PATH 中
4. 验证：
```cmd
gcc --version
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt-get update
sudo apt-get install -y gcc
```

**macOS:**
```bash
xcode-select --install
```

## 二、创建 GitHub 仓库

### 2.1 通过 GitHub API 创建（自动化方式）

```bash
# 替换 YOUR_TOKEN 为你的 GitHub Personal Access Token
curl -H "Authorization: token YOUR_TOKEN" \
  -H "Accept: application/vnd.github.v3+json" \
  https://api.github.com/user/repos \
  -d '{
    "name": "zhangyi",
    "description": "账易 - 面向个人代账会计的轻量级代理记账系统",
    "private": false,
    "auto_init": false
  }'
```

**Token 权限要求：**
- ✅ `repo` — 完整仓库访问
- ✅ `workflow` — GitHub Actions 工作流（必须！否则推不了 Actions 文件）

### 2.2 通过 GitHub 网页创建（手动方式）

1. 登录 https://github.com
2. 右上角 "+" → "New repository"
3. 填写：
   - Repository name: `zhangyi`
   - Description: `账易 - 面向个人代账会计的轻量级代理记账系统`
   - Public（公开）
   - **不要**勾选 "Add a README file"（我们要从本地推送）
4. 点击 "Create repository"

## 四、核心文件说明

### 4.1 go.mod — Go 模块定义

```go
module github.com/sevenclockseven/zhangyi

go 1.21

require (
    github.com/gin-gonic/gin v1.9.1
    gorm.io/gorm v1.25.5
    gorm.io/driver/sqlite v1.5.4
)
```

**说明：**
- `module` 路径与 GitHub 仓库地址一致
- `go 1.21` 最低 Go 版本要求
- 三个核心依赖：Gin（HTTP框架）、GORM（ORM）、SQLite驱动

### 4.2 cmd/server/main.go — 程序入口

```go
package main

import (
    "embed"
    "io/fs"
    "log"
    "net/http"
    "os"

    "github.com/gin-gonic/gin"
    "gorm.io/driver/sqlite"
    "gorm.io/gorm"

    "github.com/sevenclockseven/zhangyi/internal/api"
    "github.com/sevenclockseven/zhangyi/internal/models"
)

//go:embed all:web/dist
var webDist embed.FS
```

**关键点：**
- `//go:embed all:web/dist` — Go 的嵌入文件功能，把前端构建产物打包进二进制文件
- 启动时自动创建 `data/` 目录和 SQLite 数据库
- 自动执行数据库迁移（建表）
- 前端通过 `embed.FS` 提供静态文件服务
- SPA 路由回退：所有非 API 请求都返回 `index.html`

### 4.3 internal/models/models.go — 数据模型

定义了 10 张核心表：

| 模型 | 表名 | 说明 |
|------|------|------|
| AccountBook | account_books | 账套表 |
| Account | accounts | 科目表 |
| Voucher | vouchers | 凭证表 |
| VoucherItem | voucher_items | 凭证明细表 |
| OpeningBalance | opening_balances | 期初余额表 |
| AccountBalance | account_balances | 科目余额表 |
| AuxItem | aux_items | 辅助核算项表 |
| VoucherTemplate | voucher_templates | 凭证模板表 |
| ReportTemplate | report_templates | 报表模板表 |
| OperationLog | operation_logs | 操作日志表 |

**GORM 标签说明：**
- `gorm:"primaryKey"` — 主键
- `gorm:"index"` — 创建索引
- `gorm:"uniqueIndex"` — 唯一索引
- `gorm:"size:100"` — 字段长度
- `gorm:"default:active"` — 默认值
- `gorm:"type:decimal(14,2)"` — 精确数值类型
- `gorm:"foreignKey:BookID"` — 外键关联

### 4.5 web/vite.config.js — 前端构建配置

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  plugins: [
    vue(),
    AutoImport({ resolvers: [ElementPlusResolver()] }),
    Components({ resolvers: [ElementPlusResolver()] }),
  ],
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
      },
    },
  },
})
```

**关键点：**
- `unplugin-auto-import` — Element Plus 组件自动导入
- `unplugin-vue-components` — Element Plus 组件自动注册
- `server.proxy` — 开发时代理 API 请求到 Go 后端

### 4.6 web/src/App.vue — 布局组件

采用 Element Plus 的 Container 布局：
- 左侧 Aside：导航菜单（工作台/账套/凭证/账簿/报表/设置）
- 顶部 Header：面包屑导航
- Main：路由视图

## 六、GitHub Actions 配置

### 6.1 工作流文件 `.github/workflows/release.yml`

**触发条件：**
- 推送 `v*` tag 时自动触发（如 `v0.1.0`、`v1.0.0`）
- 支持手动触发（`workflow_dispatch`）

**构建流程：**

```
┌─────────────────┐
│  build-frontend  │  Node.js 20, npm ci, npm run build
│  (前端构建)       │  产物: web/dist/
└────────┬────────┘
         │ 上传 artifact
         ▼
┌─────────────────┐
│  build-backend   │  Go 1.21, CGO, 交叉编译
│  (后端构建)       │  5 个平台并行构建
└────────┬────────┘
         │ 上传 artifact
         ▼
┌─────────────────┐
│    release       │  下载所有 artifact
│  (发布 Release)  │  创建 GitHub Release
└─────────────────┘
```

**构建目标平台：**

| 平台 | 架构 | 输出文件 |
|------|------|----------|
| Windows | amd64 | zhangyi-windows-amd64.exe |
| Linux | amd64 | zhangyi-linux-amd64 |
| Linux | arm64 | zhangyi-linux-arm64 |
| macOS | amd64 | zhangyi-macos-amd64 |
| macOS | arm64 | zhangyi-macos-arm64 |

### 6.2 交叉编译说明

Go 的交叉编译通过环境变量控制：
```bash
GOOS=windows GOARCH=amd64 go build -o zhangyi.exe ./cmd/server/
GOOS=linux   GOARCH=amd64 go build -o zhangyi     ./cmd/server/
GOOS=darwin  GOARCH=arm64 go build -o zhangyi     ./cmd/server/
```

由于使用了 SQLite（CGO），需要安装目标平台的 C 编译器：
- Windows: `gcc-mingw-w64`
- Linux ARM64: `gcc-aarch64-linux-gnu`
- macOS: 系统自带 Xcode Command Line Tools

### 6.3 发布流程

```bash
# 1. 确保代码已提交
git add -A
git commit -m "feat: 新功能描述"

# 2. 推送到 GitHub
git push origin main

# 3. 打 tag
git tag v0.1.0

# 4. 推送 tag，触发 GitHub Actions
git push origin v0.1.0
```

推送 tag 后：
1. GitHub Actions 自动启动
2. 构建前端（~2分钟）
3. 并行构建 5 个平台的后端（~5分钟）
4. 自动创建 Release 页面
5. 5 个可执行文件作为 Release Assets 可供下载

// 替换 SQLite

## 十、常见问题

### Q1: `go build` 报错 `gcc: exec: "gcc": executable file not found`

**原因：** SQLite 需要 CGO，缺少 GCC 编译器。

**解决：**
- Windows: 安装 MinGW-w64 或 TDM-GCC
- Linux: `sudo apt-get install gcc`
- macOS: `xcode-select --install`

### Q2: `npm install` 报错 `ERESOLVE unable to resolve dependency tree`

**原因：** npm 版本过高，依赖冲突。

**解决：**
```bash
npm install --legacy-peer-deps
```

### Q3: GitHub Actions 构建失败，提示 `refusing to allow a Personal Access Token to create or update workflow`

**原因：** GitHub Token 缺少 `workflow` 权限。

**解决：**
1. 访问 https://github.com/settings/tokens
2. 编辑你的 Token
3. 勾选 `workflow` scope
4. 重新推送

### Q4: 前端页面空白，控制台报 404

**原因：** 后端未启动，或 API 代理配置错误。

**解决：**
1. 确认后端已启动：`curl http://localhost:8080/api/health`
2. 检查 `vite.config.js` 中的 proxy 配置

### Q5: 数据库文件在哪里？

SQLite 数据库文件在程序运行目录下的 `data/zhangyi.db`。

**备份：** 直接复制这个文件即可。
**恢复：** 把备份文件放回 `data/zhangyi.db`，重启程序。
